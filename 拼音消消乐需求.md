# 汉字消消乐功能需求文档

## 1. 文档信息
- 产品名称：汉字消消乐
- 站点地址：https://cn-word-game-app.netlify.app/
- 文档目的：基于线上可访问页面与前端逻辑，梳理完整功能需求，作为后续开发/测试/验收依据。
- 文档时间：2026-03-29

## 2. 产品定位与目标
- 面向用户：小学一年级到六年级学生。
- 核心目标：通过“汉字-拼音配对消除”提升识字与拼音对应能力。
- 产品承诺：纯本地学习、无广告、不收集个人信息。

## 3. 业务范围

### 3.1 教材范围
- 教材版本：人教版。
- 学段册别：
	- 一年级上册（4单元）
	- 一年级下册（8单元）
	- 二年级上册（8单元）
	- 二年级下册（8单元）
	- 三年级上册（8单元）
	- 三年级下册（8单元）
	- 四年级上册（8单元）
	- 四年级下册（8单元）
	- 五年级上册（8单元）
	- 五年级下册（8单元）
	- 六年级上册（8单元）
	- 六年级下册（8单元）

### 3.2 内容组织
- 最小学习内容：字词条（汉字 + 对应拼音）。
- 单关词量：最多12个字词条。
- 每关卡片数量：24张（12张汉字 + 12张拼音）。
- 每单元关卡数：按 `ceil(单元词量 / 12)` 自动计算。

### 3.3 复习机制
- 当某关可用新词不足12个时，系统从该单元前序词条中随机补足。
- 补足词标记为复习词（isReview=true），但在UI中与新词统一呈现。

## 4. 页面与路由需求

### 4.1 首页（年级册别选择）
- 路由：`/`
- 展示内容：
	- 品牌区：`📚 人教版`、标题`汉字消消乐`、副标题`专为小学一年级到六年级学生定制`
	- 册别卡片：12张（按教材册别动态生成，上/下册图标区分）
	- 每张卡片展示：已解锁关卡数 / 总关卡数
	- 页脚文案：`每日15分钟，轻松认识100字`、`纯本地学习，无广告，不收集信息`
- 交互：点击册别卡片进入对应单元页。

### 4.2 单元选择页
- 路由：`/unit/:grade`
- 展示内容：
	- 返回按钮
	- 标题：`{年级册别} - 选择单元`
	- 单元卡列表（按册别配置自动生成）
- 单元状态规则：
	- 单元1默认可玩
	- 单元N（N>1）需“前一单元全部关卡完成”后解锁
	- 状态显示：`🎮 可玩` / `🔒 锁定` / `✅ 已完成`
- 交互：
	- 点击可玩单元进入关卡页
	- 点击锁定单元无跳转

### 4.3 关卡选择页
- 路由：`/level/:grade/:unit`
- 展示内容：
	- 返回按钮
	- 标题：`{年级册别} - 单元{N}`
	- 关卡卡片列表（关卡序号按全册累计显示）
	- 已完成关卡显示：五星评分（填充星）+ 最佳耗时
	- 未完成关卡显示状态图标：`🎮`或`🔒`
- 关卡解锁规则：
	- 全局第1关默认可玩
	- 其他关卡需上一关已完成（跨单元连续）
- 交互：
	- 点击可玩关卡进入游戏页
	- 点击锁定关卡无跳转

### 4.4 游戏页
- 路由：`/game/:grade/:unit/:level`
- 头部区域：
	- 返回按钮
	- 当前关卡编号（全册累计）
	- 计时器（`mm:ss`）
- 主区域：
	- 24宫格卡片
	- 卡片类型：`char`（汉字）/`pinyin`（拼音）
	- 卡片状态：selected、matched、hidden、mismatch
- 底部区域：
	- 已消除进度：`已消除: x/24`
	- 进度条：按已消除比例实时更新
	- 控制按钮：`💡 提示`、`🔊 声音`（可切换`🔇 静音`）

### 4.5 通关页
- 路由：`/victory/:grade/:unit/:level`
- 展示内容：
	- 奖杯图标与`恭喜通关！`
	- 关卡描述：`{年级册别} 单元{N} 第{全册关卡号}关`
	- 成绩区：五星 + 最佳耗时
	- 本关生字列表（拼音 + 汉字）
	- 按钮：`下一关`（存在下一关时显示）、`返回关卡列表`
- 下一关规则：
	- 若本单元后续关卡存在，跳转下一关
	- 否则若下一单元存在关卡，跳转下一单元第1关

## 5. 核心玩法需求

### 5.1 基本操作
- 用户每次最多选中2张未匹配卡片。
- 两张卡片满足“汉字与其拼音互为对应”即判定成功。
- 成功配对后：
	- 播放正确音效（若声音开启）
	- 约500ms后置为matched并隐藏
	- 已消除数量+2
- 错误配对后：
	- 播放错误音效（若声音开启）
	- 错误次数+1
	- 卡片短暂标记mismatch（约800ms）后取消选中

### 5.2 关卡完成条件
- 当已消除数量达到总卡片数（24）即完成关卡。
- 完成后约1秒自动跳转通关页。

### 5.3 提示功能
- 点击`💡 提示`时：
	- 自动选中一组可配对的“汉字+拼音”
	- 高亮约1秒后取消
	- 记为一次错误（错误次数+1）
- 若无可配对项（已完成）则不执行。

### 5.4 声音功能
- 支持全局开关。
- 开启时：
	- 正确配对播放高频短促提示音
	- 错误配对播放低频提示音
- 关闭时：不播放任何音效。

## 6. 评分与统计需求

### 6.1 星级规则
- 以错误次数计算星级：
	- 错误=0：5星
	- 错误≤2：4星
	- 错误≤4：3星
	- 其他：2星

### 6.2 最佳成绩覆盖规则
- 每关保存最佳成绩：`errors`、`time`、`stars`。
- 仅在以下任一条件满足时覆盖历史记录：
	- 新星级更高
	- 星级不变但耗时更短

### 6.3 时间显示规则
- 游戏内实时显示：`mm:ss`
- 通关页历史成绩显示：`mm分ss秒`

## 7. 学习进度与解锁需求

### 7.1 存储方式
- 仅使用浏览器本地存储（localStorage）。
- 存储键：`game_progress`
- 不依赖后端账号体系，不发生云端同步。

### 7.2 进度数据结构
- 基础字段：
	- `version`：进度版本（当前`4.0`）
	- `currentGrade`、`currentUnit`、`currentLevel`
	- `unlockedLevels`：按册别保存已完成关卡key列表（`unit-level`）
	- `levelStats`：按册别/关卡存储最佳成绩
	- `settings.sound`：声音开关

### 7.3 首页进度计算
- 每册显示：`已完成(解锁)关卡数 / 总关卡数`
- 进度条百分比：`round(已完成关卡 / 总关卡 * 100)`

### 7.4 版本迁移
- 当读取到旧版本进度时，自动迁移到当前结构并覆盖保存。

## 8. 非功能需求

### 8.1 隐私与安全
- 不采集个人信息。
- 不接入广告SDK。
- 不要求登录。

### 8.2 可用性
- 支持浏览器直接访问。
- 页面需保持返回链路清晰（首页→单元→关卡→游戏→通关）。

### 8.3 性能
- 关卡内卡片生成与洗牌应在页面加载时快速完成。
- 计时与动画交互流畅，无明显卡顿。

## 9. 业务流程（端到端）
1. 用户进入首页选择年级册别。
2. 进入单元页，选择已解锁单元。
3. 进入关卡页，选择已解锁关卡。
4. 在游戏页进行汉字-拼音配对，系统实时计时与计错。
5. 全部配对后系统保存成绩、更新解锁状态并跳转通关页。
6. 用户可继续下一关或返回关卡列表复习。

## 10. 验收清单
- 首页可正确展示4个册别及各自进度。
- 单元锁定规则生效：未完成前序单元时不可进入后续单元。
- 关卡锁定规则生效：未完成上一关不可进入下一关。
- 正确/错误配对行为、时序与UI状态符合规则。
- 提示功能会高亮一组可配对项并计入错误。
- 声音开关可实时切换并影响后续音效。
- 通关后可看到星级、耗时、本关生字与跳转按钮。
- localStorage中可看到`game_progress`并含进度、统计、设置字段。
- 刷新页面后进度与成绩可恢复。

## 11. 当前实现边界说明
- 当前为纯前端单机学习模式，不含：
	- 账号系统
	- 多端同步
	- 排行榜
	- 社交分享
	- 后台内容管理系统

## 12. 术语与编号规则
- 关卡序号：分为“单元内关卡序号”和“全册累计关卡序号”，页面展示采用全册累计关卡序号。
- 完成：指该关卡完成并写入进度存储。
- 解锁：指关卡/单元允许进入，并不等同于完成。
- 最佳成绩：同一关卡下按“星级优先、同星级比耗时”保留唯一记录。

## 13. 异常与边界处理需求
- 路由参数非法：当 `grade`、`unit`、`level` 越界或格式错误时，应重定向到最近合法页面（优先返回上一层列表页）。
- 数据缺失：当词库不存在对应单元或关卡时，页面需展示空状态并提供返回入口，不得白屏。
- 本地存储异常：
	- localStorage 读取失败或 JSON 解析失败时，回退默认进度。
	- 保存失败时不阻塞主流程，需记录可观测日志（console warning 级别）。
- 计时器一致性：离开游戏页时必须销毁计时器，避免重复计时与内存泄漏。
- 连续点击保护：匹配判定执行窗口内应屏蔽额外点击，防止状态错乱。

## 14. 兼容性与可访问性需求
- 终端适配：支持手机与桌面浏览器，核心操作在 360px 及以上宽度可用。
- 浏览器兼容：支持近两年主流 Chromium 内核浏览器。
- 触控友好：卡片点击热区不小于 40px。
- 可访问性：
	- 关键按钮需有可识别文本（返回、提示、声音、下一关）。
	- 色彩状态（匹配/错误）应辅以形态变化，不仅依赖颜色。

## 15. 内容与数据治理需求
- 词库正确性：每个词条必须满足“一汉字 + 一拼音”映射，禁止重复冲突映射。
- 内容版本化：词库调整时需记录版本号，并与进度版本分离管理。
- 文案规范：学习相关文案保持低年级可读性，避免专业术语堆叠。

## 16. 发布与回归要求
- 发布前必须完成：
	- 关键路径冒烟：首页 -> 单元 -> 关卡 -> 游戏 -> 通关。
	- 进度回归：刷新后进度可恢复，版本迁移可执行。
	- 解锁回归：跨关卡、跨单元解锁链路正确。
- 部署要求：静态托管需配置 SPA 路由回退到首页，避免子路径直访 404。

## 17. 验收补充指标（量化）
- 首屏可交互时间目标：普通网络环境下 <= 3s。
- 页面切换响应目标：普通设备上 <= 200ms（不含首次资源加载）。
- 稳定性目标：连续完成 20 关无卡死、无白屏、无进度丢失。

## 19. 实施前待确认事项
- 音效策略：iOS/部分浏览器对自动播放限制较严格，是否接受“首次用户点击后才启用音频上下文”。
- 浏览器范围：是否仅保障 Chromium 内核，还是额外覆盖 Safari 桌面版。
- 词库维护方式：短期是否采用手工维护静态词库，或需要引入词库校验脚本。
- 文案风格：是否需要提供“简体中文 + 拼音注音一致性检查”规范。
- 性能目标口径：首屏与切页指标采用开发机还是中低端设备作为验收基准。
