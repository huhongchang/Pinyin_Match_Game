## 3. 总体技术架构

### 3.1 架构形态
- 应用形态：前端 SPA
- 数据形态：静态词库 + 本地进度存储
- 部署形态：静态站点托管（Netlify/Vercel/GitHub Pages 均可）

### 3.2 分层设计
- 展示层（UI）：页面、组件、交互动画
- 路由层（Router）：页面切换、参数校验、非法路径回退
- 领域层（Domain）：关卡构建、匹配判定、解锁计算、评分规则
- 数据层（Data）：词库读取、本地存储读写与迁移
- 基础能力层（Core）：音效、时间格式化、洗牌算法、日志

### 3.3 模块边界原则
- 规则相关逻辑集中在领域层，不散落在组件中。
- 本地存储统一经 Data 层访问，不在组件中直接读写 localStorage。
- 页面仅处理状态渲染与用户事件，不直接持有复杂业务判断。

## 4. 技术选型建议
- 框架：Vue 3 + TypeScript
- 路由：Vue Router
- 状态管理：Pinia
- 构建工具：Vite
- 样式：CSS Modules 或 SCSS（二选一，保持一致）
- 单元测试：Vitest
- E2E：Playwright
- 代码质量：ESLint + Prettier

选型原因：
- Vue 3 组合式 API 对页面型应用结构清晰。
- TypeScript 降低规则逻辑出错率。
- Vite 构建快，适合迭代频繁的小型教育项目。

## 5. 数据设计

### 5.1 词库模型
- WordItem
  - char: string
  - pinyin: string
- Grade -> Unit -> WordItem[]

约束：
- 每个词条必须存在且仅存在 1 对合法映射。
- 同一关卡内不允许出现冲突映射。

### 5.2 进度模型
- Progress
  - version: string
  - currentGrade: string
  - currentUnit: number
  - currentLevel: number
  - unlockedLevels: Record<string, string[]>
  - levelStats: Record<string, Record<string, LevelStat>>
  - settings: { sound: boolean }
- LevelStat
  - errors: number
  - time: number
  - stars: number

### 5.3 存储策略
- 存储键：game_progress
- 存储介质：localStorage
- 异常策略：读取失败回退默认值，保存失败只告警不阻塞流程。
- 版本迁移：按 version 字段执行迁移函数，迁移成功后覆盖回写。

## 6. 核心流程设计

### 6.1 关卡生成流程
1. 按册别 + 单元 + 关卡切片获取本关主词。
2. 若不足 12 词，从同单元历史词随机补足。
3. 生成 24 张卡片（汉字/拼音各 12）。
4. 使用 Fisher-Yates 洗牌。

### 6.2 游戏交互流程
1. 玩家点击卡片，最多选中 2 张未匹配卡。
2. 若两张互为映射，延时标记 matched + hidden。
3. 若不匹配，记录错误并在短暂高亮后回退。
4. 当 matched 数量达到总卡数，触发通关。

### 6.3 通关收敛流程
1. 停止计时器。
2. 计算 stars 并更新最佳成绩。
3. 更新 unlockedLevels。
4. 持久化进度。
5. 跳转通关页。

### 6.4 提示与音效流程
- 提示：寻找一组可配对项 -> 临时高亮 -> 计错一次。
- 音效：
  - 正确匹配：高频短促音
  - 错误匹配：低频提示音
  - 静音开关：全局控制是否播放

## 7. 状态机设计

建议将游戏态拆分为：
- Ready：初始可点击
- OneSelected：已选一张
- Resolving：正在判定，不允许新点击
- IdleAfterResolve：判定结束回到可点击
- Finished：通关

收益：
- 可防止连点导致的竞态问题
- 可减少组件条件分支复杂度
- 测试用例可基于状态迁移覆盖

## 8. 路由与部署设计

### 8.1 路由规则
- /
- /unit/:grade
- /level/:grade/:unit
- /game/:grade/:unit/:level
- /victory/:grade/:unit/:level

### 8.2 参数校验
- grade 必须在允许列表内。
- unit/level 必须为正整数且不超过上限。
- 非法参数回退上一层列表页，避免白屏。

### 8.3 部署注意事项
- 必须开启 SPA 回退重写到首页。
- 资源路径推荐相对路径或基于站点根路径构建。

## 10. 性能与稳定性目标
- 首屏可交互 <= 3s
- 页面切换响应 <= 200ms
- 连续完成 20 关无白屏、无进度丢失

优化建议：
- 卡片动画优先使用 transform/opacity
- 减少不必要响应式依赖
- 计时器仅在游戏页生命周期内存活

## 12. 风险与预案
- 风险：SPA 子路径直访 404
  - 预案：部署配置 rewrite 到首页
- 风险：localStorage 受限或损坏

- 风险：词库更新导致旧进度错位
  - 预案：维护词库版本并提供映射迁移策略
