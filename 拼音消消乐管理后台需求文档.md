# 拼音消消乐管理后台需求文档

## 1. 文档信息
- 文档名称：拼音消消乐管理后台需求文档
- 文档版本：V1.0（评审草案）
- 文档日期：2026-03-31
- 适用范围：管理后台产品需求定义，不包含编码实现
- 关联文档：拼音消消乐需求.md、拼音消消乐技术文档.md、拼音消消乐开发任务拆解.md

## 2. 背景与目标

### 2.1 背景
当前产品侧缺少统一的数据可视化入口，无法快速判断：
- 日常使用规模是否增长
- 学习链路在哪些环节流失
- 哪些学科、年级、单元、关卡存在明显问题

### 2.2 目标
建设一个简单、稳定、可运营的数据管理后台，支持运营和产品按日常节奏查看趋势、定位问题、导出数据。

### 2.3 成功标准
- 进入后台后，3 分钟内可完成核心经营面判断（规模、转化、学习效果）。
- 指标定义一致，页面展示与导出口径一致。
- 不采集个人敏感信息，符合现有产品隐私承诺。

## 3. 范围定义

### 3.1 本期范围（MVP）
1. 后台简单账号登录。
2. 总览页（核心指标卡 + 趋势图 + 学科占比）。
3. 访问分析（页面访问、停留、跳出）。
4. 学习漏斗（首页到通关全链路）。
5. 学习效果（开始、完成、用时、错误、提示）。
6. 时间与维度筛选。
7. 数据导出（CSV）。

### 3.2 本期不做
1. 多租户与组织架构管理。
2. 秒级实时监控。
3. 自定义图表拖拽平台。
4. 自动告警系统（短信/电话/机器人）。


## 4. 账号方案（简化版）
1. 本期采用简单账号登录，不做角色与权限体系。
2. 默认支持 1 组后台账号（账号名 + 密码）。
3. 所有登录用户看到相同页面和数据范围。
4. 暂不支持细粒度权限、菜单权限、字段权限。



## 5. 关键业务问题
后台必须回答以下问题：
1. 今天和近 7/30 天有多少有效使用。
2. 从首页到开始游戏、到完成通关的转化率是多少。
3. 语文/数学/英语三科占比和趋势如何。
4. 哪些年级、单元、关卡完成率较低。
5. 平均用时和错误次数是否异常。

## 6. 功能需求

### 6.1 登录与会话管理
1. 支持账号密码登录。
2. 未登录访问后台页面时跳转登录页。
3. 支持手动退出登录。
4. 登录态过期后需重新登录。
5. 登录失败需提示原因（账号不存在或密码错误）。

### 6.2 总览页
#### 6.2.1 指标卡
- 活跃设备数（匿名去重）
- 会话数
- 学习开始次数
- 通关次数
- 平均会话时长

#### 6.2.2 趋势图
- 近 7 天趋势
- 近 30 天趋势
- 支持切换展示维度（活跃、开始、完成）

#### 6.2.3 结构占比
- 学科占比：语文、数学、英语
- 可切换按开始次数或通关次数统计

### 6.3 访问分析
1. 页面访问量排行（PV）。
2. 页面独立访问排行（UV，匿名设备去重）。
3. 平均停留时长。
4. 跳出率。
5. 入口页与退出页分布。

### 6.4 学习漏斗分析
漏斗固定节点：
1. 首页曝光
2. 进入学科页
3. 进入单元页
4. 进入关卡页
5. 开始游戏
6. 完成通关

每个节点展示：
- 到达量
- 节点转化率
- 节点流失率

### 6.5 学习效果分析
1. 游戏开始次数。
2. 通关完成次数。
3. 首次通关率。
4. 平均通关用时。
5. 平均错误次数。
6. 提示使用率。

### 6.6 筛选与对比
1. 时间筛选：今天、昨天、近 7 天、近 30 天、自定义。
2. 维度筛选：学科、年级、单元、关卡。
3. 对比方式：与上一周期对比（环比）。

### 6.7 数据导出
1. 导出范围与当前筛选一致。
2. 导出格式：CSV。
3. 导出文件必须包含：导出时间、筛选条件、指标字段说明。

## 7. 指标口径定义

### 7.1 基础定义
- 活跃设备：统计周期内至少触发一次有效事件的匿名设备数。
- 会话：同一设备连续 30 分钟无行为则会话结束。
- 学习开始：触发 game_start 事件计 1 次。
- 通关完成：触发 game_complete 事件计 1 次。

### 7.2 关键公式
- 漏斗转化率 = 当前节点到达量 / 上一节点到达量
- 漏斗流失率 = 1 - 漏斗转化率
- 跳出率 = 仅访问一个页面的会话数 / 总会话数
- 首次通关率 = 首次完成关卡人数 / 首次开始关卡人数

## 8. 数据采集需求（埋点事件）

| 事件名 | 触发时机 | 必填属性 |
|---|---|---|
| page_view | 页面展示 | page_name, path, timestamp |
| subject_enter | 进入学科 | subject |
| unit_enter | 进入单元 | subject, grade, unit |
| level_enter | 进入关卡列表 | subject, grade, unit, level |
| game_start | 开始游戏 | subject, grade, unit, level |
| game_match_success | 成功配对 | subject, level |
| game_match_fail | 失败配对 | subject, level |
| game_hint_use | 使用提示 | subject, level |
| game_complete | 完成通关 | subject, grade, unit, level, duration, error_count |
| page_leave | 页面离开 | page_name, stay_duration |

约束要求：
1. 事件必须带匿名设备标识（不可逆、不可识别个人）。
2. 禁止上传姓名、手机号、身份证等个人敏感信息。
3. 事件时间使用统一时区与时间格式。

## 9. 后台页面信息架构
1. 登录页
2. 总览页
3. 访问分析页
4. 学习漏斗页
5. 学习效果页
6. 导出记录页（可选，MVP 建议保留基础列表）

## 10. 非功能需求

### 10.1 性能
- 首屏加载时间小于等于 3 秒（常规网络环境）。
- 普通筛选查询响应时间小于等于 2 秒。

### 10.2 可用性
- 后台可用性不低于 99.5%。

### 10.3 安全
1. 后台接口必须鉴权。
2. 账号密码需加密存储，不允许明文保存。
3. 导出行为需记录审计日志（时间、筛选条件）。

### 10.4 兼容性
- 优先支持主流 Chromium 浏览器。

## 11. 隐私与合规
1. 延续产品“无个人信息收集”原则。
2. 数据展示以聚合统计为主，不提供可识别单人行为查询。
3. 数据默认保留 180 天，超期自动清理（可配置）。

## 12. 验收标准（DoD）
1. 简单账号登录可用，未登录不可访问后台页面。
2. 总览、访问、漏斗、学习效果四类报表可稳定展示。
3. 时间与维度筛选正确联动，结果与口径定义一致。
4. 导出文件内容与页面展示一致。
5. 抽检事件不包含个人敏感信息字段。

## 13. 里程碑建议
- 第 1 周：口径冻结 + 埋点清单评审。
- 第 2 周：后台页面原型与字段评审。
- 第 3 周：联调验证与数据对齐。
- 第 4 周：验收测试与上线准备。

## 14. 风险与应对
1. 口径分歧：提前冻结指标定义，变更走评审流程。
2. 埋点缺失：上线前执行埋点覆盖清单检查。
3. 查询性能波动：先聚合后明细，必要时增加缓存。
4. 隐私风险：建立字段白名单与发布前扫描机制。

## 15. 待确认项
1. 后台账号采用单账号还是多账号（均不做权限区分）。
2. 是否需要周报邮件（若需要，建议列入下一期）。
3. 是否需要按设备类型、地区维度做分层统计。
4. 数据导出是否需要审批流。
