一、推荐方式：推送后自动更新（最省心）
1. 先做一次性确认（只需检查一次）
1. Netlify 站点已连接你的 GitHub 仓库
2. Production branch 是 main
3. 构建配置正确（你项目里已经有配置）：netlify.toml

2. 以后每次更新线上
1. 本地改代码
2. 提交并推送到 main
3. Netlify 会自动触发构建并发布到线上
4. 到 Netlify 后台 Deploys 页面看状态，变成 Published 就是上线成功

二、手动强制更新（自动部署失败时用）
1. 本地先构建一次，确保无报错
命令：npm run build

2. 直接发布到生产
命令：netlify deploy --prod --dir=dist --functions=netlify/functions --message "你的发布说明"

3. 发布后刷新线上域名确认

三、我建议你固定成这个习惯
1. 日常走自动部署（push main 即上线）
2. 只有在紧急修复或自动构建异常时，才用手动 deploy --prod
3. 每次发布后，至少检查两件事：
1. 首页是否正常打开
2. 管理后台报表接口是否能返回数据

四、常见失败排查（简版）
1. 构建失败
先在本地跑 npm run build，看报错再修

2. 线上还是旧版本
确认你推送的是 main，且 Netlify 对应站点连接的是这个仓库

3. 接口 404 或函数不生效
手动部署时要带 --functions=netlify/functions
